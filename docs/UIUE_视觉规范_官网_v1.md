# 元都AI 官网 UI/UE 视觉规范（V1.0）

> 适用范围：`frontweb` 官网与用户中心（含原创表情、我的上传、投稿审核）  
> 最近更新：2026-04-06  
> 目标读者：产品经理、UI 设计师、前端工程师、运营

---

## 1. 设计目标与体验原则

### 1.1 设计目标
- **专业可信**：体现 AI 视觉资产平台的技术感与可靠感。
- **操作高效**：上传、提审、审核、上架链路清晰，降低理解成本。
- **内容优先**：表情包素材卡片展示优先，操作控件辅助不喧宾夺主。

### 1.2 体验原则
1. **主路径优先**：所有页面优先展示“下一步动作”。
2. **状态可见**：审核状态、上架状态、错误与成功必须即时反馈。
3. **低学习成本**：同类卡片、按钮、筛选器在全站一致。
4. **可运营扩展**：后台审核台支持批量处理和日志追溯。

## 3. 视觉语言规范

### 3.1 品牌调性关键词
- 清爽、现代、可信、轻科技
- 高留白 + 弱分割 + 圆角卡片

### 3.2 色彩系统（推荐）
- **品牌主色（强调）**：Emerald 500/600  
  - 用途：主按钮 hover、激活态、状态正向反馈
- **深色主按钮**：Slate 900  
  - 用途：关键动作（新建合集、提交、确认）
- **中性色**：Slate/Zinc 50~900  
  - 用途：背景、文字、边框、分割
- **状态色**
  - 成功：Emerald
  - 警示：Amber
  - 错误：Rose
  - 信息：Slate

### 3.3 排版规范
- 中文优先字体栈：PingFang SC / Hiragino Sans GB / Microsoft YaHei
- 标题层级
  - H1：`text-3xl font-black`
  - H2：`text-2xl font-black`
  - 卡片标题：`text-base font-black`
- 正文：`text-sm` 为主
- 辅助信息：`text-xs` 或 `text-[11px]`

### 3.4 圆角与阴影
- 主卡片：`rounded-[1.75rem]` 或 `rounded-2xl`
- 按钮：`rounded-xl`
- 小标签：`rounded-full`
- 阴影：默认 `shadow-sm`，hover `shadow-lg`

### 3.5 间距系统
- 页面外边距：`max-w-6xl / max-w-7xl + px-6 + py-10`
- 区块间距：`space-y-6 / space-y-8`
- 卡片内边距：`p-4 / p-5`

## 4. 组件规范（UI）

### 4.1 按钮
- 主按钮：深色底白字，hover 过渡到品牌色
- 次按钮：浅底边框
- 危险按钮：Rose 边框+文字
- 禁用态：`disabled:opacity-40~60`

### 4.2 状态标签（Badge）
- 审核中：Amber
- 审核通过：Emerald
- 驳回：Rose
- 草稿/下架：Slate

### 4.3 卡片（合集卡）
- 封面区：固定比例 `aspect-[4/3]`
- 信息区：标题、描述、数量、更新时间、操作
- Hover：轻微上浮 + 放大封面 + 边框高亮

### 4.4 表单
- 输入框高度统一（`h-9/h-11`）
- 聚焦态使用品牌色边框
- 错误文案靠近字段下方出现

### 4.5 反馈
- 错误提示：顶部条状浅红底（可消失）
- 成功提示：可选 toast/行内提示
- 加载：局部按钮 spinner + 全页 skeleton/中心 spinner

## 5. 关键页面模板（UE）

### 5.1 我的上传（操作）
- 目标：创建、编辑、删除、上传素材
- 强调：素材管理效率
- 操作：新建合集、管理表情、编辑元数据

### 5.2 投稿审核（操作）
- 目标：从内容管理过渡到审核与上架
- 关键动作：
  - 提交审核
  - 上架
  - 下架
- 必须展示：
  - 审核状态
  - 上架状态
  - 驳回原因/下架原因
  - 提审次数与时间

### 5.3 表情包作品（展示）
- 目标：展示用户作品，不做编辑行为
- 入口：我的作品 > 表情包作品
- 行为：点击合集查看详情

### 5.4 原创表情（官网公共页）
- 目标：展示通过审核并上架的 UGC 合集
- 数据约束：`source=ugc_upload && approved && online`

### 5.5 管理后台 UGC Reviews
- 目标：运营高效审核
- 支持：
  - 单条审核（通过/驳回/下架）
  - 批量操作
  - 审核日志追溯

## 6. 交互与动效规范

### 6.1 动效时长
- hover/focus：`200~300ms`
- 页面级切换：尽量弱化（避免复杂转场）

### 6.2 可点击反馈
- 所有可点击元素必须有 hover 与 disabled 视觉差异
- 菜单下拉支持 hover + click 双触发

### 6.3 状态一致性
- 所有审核/上架动作后，列表应即时刷新
- 错误要明确告诉用户下一步（重试/修改后提审）

---

## 7. 响应式规范

- 断点建议
  - `sm`：2 列
  - `md`：2~3 列
  - `lg/xl`：3~4 列
- 移动端优先保留：
  - 关键状态标签
  - 主操作按钮
  - 核心标题与更新时间

---

## 8. 无障碍（A11y）规范

- 图像必须有 `alt`
- 按钮必须有可读文案（仅图标按钮需 `title/aria-label`）
- 颜色不是唯一信息来源（状态+文字双表达）
- 键盘可访问：Tab 顺序按视觉流程

---

## 9. 文案规范（推荐）

- 状态词统一：
  - 草稿 / 审核中 / 审核通过 / 已驳回 / 已上架 / 已下架
- 操作文案统一：
  - 提交审核 / 上架 / 下架 / 查看详情 / 去管理
- 错误文案结构：
  - “动作 + 失败原因 + 建议下一步”
  - 例：`提交审核失败，请补充至少 1 张表情后重试`
